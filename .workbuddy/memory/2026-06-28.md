# 2026-06-28 工作日志

## 脱敏处理：将敏感配置从源代码提取到独立配置文件

### 背景
项目中多个网络层文件硬编码了敏感信息（AES 密钥/IV、签名密钥、API 地址）。

### 处理内容
1. **新增 `secrets.properties`** — 集中存放所有敏感配置（AES_KEY, AES_IV, SIGN_KEY, BASE_URL, API 版本号）
2. **新增 `secrets.properties.example`** — 模板文件，作为新成员接入指引
3. **修改 `app/build.gradle.kts`** — 读取 secrets.properties，通过 `buildConfigField` 注入到 BuildConfig
4. **修改 4 个源文件**，从 `BuildConfig` 读取配置而非硬编码：
   - `AesCipher.kt` — ENCODED_KEY / ENCODED_IV
   - `ExamDecryptInterceptor.kt` — AES_KEY / AES_IV
   - `SignInterceptor.kt` — SIGN_KEY
   - `RetrofitClient.kt` — BASE_URL / ARTICLE_API_VERSION / MATERIAL_API_VERSION
5. **更新 `.gitignore`** — 添加 `/secrets.properties` 忽略规则

### 使用方式
新成员接入：`cp secrets.properties.example secrets.properties` 后填写真实值。
